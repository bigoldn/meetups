Some Quick Background

A Decision Tree is a simple predictive model consisting of binary split nodes and leaf nodes.

  1. Split Nodes: Contain a binary split function which decides which of it's two children a sample progresses to.
  2. Leaf Nodes: Contain a class prediction (either a single class or a class density).

Building Decision Trees

There are many algorithms for building decision trees (CART, C4.5), but the basic idea is usually:

  1. Choose a loss function: Gini, Entropy.

  2. Follow this basic procedure:

    def DecisionTree(X, y, loss_function):
    tree = node(X, y)
    nodes = [tree]
    while len(nodes) != 0:
        node = nodes.pop()
        if len(node.X) > 1:
            # Find the best single feature split with respect to the loss
            best_split = find_best_single_feaute_split(node.X, node.y, loss_function)
            nodeL, nodeR = node.split(best_split)
            node.left_child, node.right_child  = nodeL, nodeR
            nodes.append([nodeL, nodeR])
    return tree

This recursive procedure does not result in the optimal decision tree, but that is an NP-hard problem.

Random Forests

Decision Trees built to sufficient depth give powerful models with low bias, but have very high variance.

Bagging (or Bootstrap Aggregation) many trees is an obvious solution. Each tree identically distributed but not independent so:

  • Bias is unchanged.

  • Variance of an average of N estimators, each with variance σ and pairwise correleation ρ is

    ρσ2+1−ρNσ2

Random Forests add an extra step of including random features for splitting, which significantly decorrelates the individual estimators, reducing ρ.

Comparing Trees and Forests.

This talk will present two new (to me) applications of these ideas:

  • Isolation Forests: A random forest based algorithm for outlier detection.
  • Random Jungles: An extension of Random Forests which attempts save memory by allowing the Trees to become DAGs.

    • Isolation Forests: The Intuition

    The basic idea is to use trees for finding outliers by using a measure of isolation.

    Points in a sample are considered to be isolated if they are easy to separate from the rest of the sample.

    The Isolation Tree

    The Isolation Tree measures isolation by relating it to the depth of a point within a random Decision Tree.

    For sample X we build an Isolation Tree:

    def IsolationTree(X):
    tree = node(X)
    nodes = [tree]
    while len(nodes) != 0:
        node = nodes.pop()
        if len(node.X) > 1:
            nodeL, nodeR = node.random_split()
    return tree

    Then define:

    1. h(x): The path length of a point. This is the depth required to reach a sample point in the random tree. This depends on N.
    2. c(n)=2H(n−1)−(2(n−1)/n): The average path length of an unsuccessful search in a binary search tree.

    The point is that E(h(x))≈c(N)

    The Isolation Forest

    As with the classification application, we can build Bootstrap Aggregated Isolation Trees with random split features so that we approximate:

    E(h(x))=1/m∑hi(x)

    From this we define the anomaly score as

    a(x)=2−h(x)/c(n)

    and find that (heuristically)

    • s≈1 implies definitely outlier.
    • s≪0.5 implies safely a normal instance (we use this as the boundary).
    • ∀s≈0.5 the entire sample is fairly well distributed.

    Properties Of The Isolation Forest

    • Isolation Forests work well with small sample sizes (density measures are not used) and so subsampling can dramatically speed up and improve detection.

    • Isolation Forests are fast to build and evaluate. In many cases full depth is not required and once built they are quick to evaluate.
    • High dimensional, messy and categorical data are all easy to handle without modification of the basic procedure.

    Random Jungles

    Random Jungles extend Random Forests by allowing the binary internal nodes to share children.

    Random Jungles are ensembles of simple DAGs (directed acyclic graphs).

    Motivation: The Problem With Decision Trees

    Decision Trees are incredibly popular (especially as Random Forests) but there are some problems:

    1. Memory use: The memory requirement of decision trees increases exponentially with the average depth.
    2. Slow training: Although they can be trained in parallel, many trees can be required, and building can be slow, for large sample sizes.
    3. Shallow Trees are weak: Deep trees are required to generalization to be effective.
    4. Ensembles of many trees are hard to understand: No one really knows what is going on with their model.

    Motivation: Jungles And Memory

    Jungles can limit the growth of memory requirements while still expressing complicated relationships

    Motivation: Jungles And generalization

    Consider trying to identify a cow..

    Jungles allow for more natural expression of segmented relationships.

    Building Jungles

    The basic approach is to assume that we can build each DAG one level at a time. At each level we need to simultaneously choose the split features/values and the branching structure.

    With loss function L we define:

    1. M: The maximum number of output nodes for each
    2. fi: The split function from node i.
    3. Si: The sample reaching node i in the next level.
    4. li,ri: The right and left outputs from node i.

    Once the tree has been built to the point where M constrains the total number of outputs we must optimize

    min{fi},{li},{ri}∑jL(Sj)

    subject to having M or less distinct li and ri.

    There are a few proposed algorithms for optimizing these choices, but we'll see LSearch.

    LSearch switches between two coordinate descent, first on split-optimization and then on branch-optimization.

    def LSearch(in_nodes, loss, M):
    split_values, out_edges = initialize(in_nodes, M)
    Li = loss(in_nodes, split_values, out_edges)
    while "still improving":
        # split-optimization
        for node in in_nodes:
            split_values[node] = best_split(loss, node, in_nodes, split_values, out_edges)
        # branch optimization
        for node in in_nodes:
            out_edges[node] = best_out(loss, node, in_nodes, split_values, out_edges)
        total_loss = loss(in_nodes, split_values, out_edges)
    return split_values, out_edge

    Notes:

    1. The best_split and best_out functions require the entire set of nodes in order to optimize.
    2. The best_split and best_out may result in a node only having a single out node.
